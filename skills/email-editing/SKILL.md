# Email editing

Use this skill whenever writing or editing an email, including a Gmail draft.

## Default editing standard

Edit according to George Orwell’s principles for writing:

1. Never use a metaphor, simile, or other figure of speech which you are used to seeing in print.
2. Never use a long word where a short one will do.
3. If it is possible to cut a word out, always cut it out.
4. Never use the passive where you can use the active.
5. Never use a foreign phrase, a scientific word, or a jargon word if you can think of an everyday English equivalent.
6. Break any of these rules sooner than say anything outright barbarous.

## Practical rules

- State the purpose in the first paragraph.
- Use short paragraphs and direct requests.
- Use a real HTML `<ul>`/`<li>` list for multiple questions or actions when creating an HTML email.
- Keep relevant facts, constraints, dates, names and questions; do not silently remove a material request.
- Match the requested level of formality. If none is given, be warm, professional and concise.
- Do not invent personal details, promises, deadlines or recipient information.
- When revising an existing draft, preserve the sender’s intent and flag any ambiguity rather than guessing.

## Gmail drafts

When editing an email in Gmail, use the available Gmail/Google Workspace tooling to update the existing draft rather than creating a duplicate, unless the user asks for a new draft. Never send an email unless the user explicitly asks to send it.
